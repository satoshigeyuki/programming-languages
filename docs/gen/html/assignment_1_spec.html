Specification

You collaborate with team members. Here are responsibilities of you and your group.

  1. Each member submits your calculator program through Jupyter by pressing the "Submit" button of pl05_calculator. Make sure you have your source code in the specified directory with the specified names, so I can automatically check them (i.e., do not roll your own file names).

    Other materials should be submitted by editing collaborative documents (Microsoft Excel/Word) I give to each group. A folder named programming-languages-group-NN should appear in your OneDrive page's "Shared" folder on the left pane, where NN is your group number. The folder contains a child directory assignment-1, which contains two files record.xlsx and summary-reflection.docx

    To see these files, you have to sign in Microsoft 365 with UTokyo Account (see this page if you don't know how).

    To indicate you are able to access these files, please write your name and student id in work log sheet in record.xlsx immediately.

  2. A group has to compile records of when each member worked, both individually or with other members into work log sheet in record.xlsx.

    A group has to gather and have meetings a few times. They can/should also include meetings in classes (if and only if the group indeed met during a class).

  3. In the end, each member has to show other members your source code and if it works (compile and run it before other members). All members must see how far others went and honestly report what they saw in the check sheet in record.xlsx.

    You should select from the following.

    • no show --- the student did not appear in meetings
    • no code --- the student participated but could show no or only little code
    • compile error --- the code was made but it had syntax/static errors, so could not start execution
    • runtime error --- the code compiled so could start execution, but could not run until the end due to an error during the execution
    • wrong results --- the code could run until the end but some results were wrong
    • ok --- the code could produce correct results for all tests

  4. (The main part) In the meeting(s), exchange and discuss, among others, following topics

    • how easy or difficult was it to write it, including what was difficult or which aspect of the language you worked with made it difficult
    • how their programs could be improved for clarity, conciseness, performance, etc.
    • which aspects of each language make your language more productive or less so
    They should be written to summary-reflection.docx While I don't impose any page limit, I expect it to be two to a few pages.

  5. After you have done, each member should send a brief notice to the assignment 1 in ITC-LMS saying something like "we have done all the work specified" and optionally some nice greetings or remarks.

Notes

  • It's due May 27th (Sat)
  • You will have a few opportunities to work on it during classes, but it's not that you can finish everything during the class time. You are expected to work on it outside the class, and more importantly, meet and collaborate with and other group members.

Misconducts

It goes without saying that any of the following constitutes a misconduct.
  • dishonestly report status of your or another student's code (e.g., say "ok" when the code in fact does not work)
  • dishonestly report your or another student's participation in a meeting (e.g., record as if you were in a meeting or a lecture when you in fact weren't)
  • dishonestly report discussions or exchanges that actually did not happen
